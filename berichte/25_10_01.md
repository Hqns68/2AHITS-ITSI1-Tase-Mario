# Grundlage

Shell: ist ein Programm das eine textorientierte Schnittstelle zum betriebssysem zur Verfügung stellt.

Prompt:

```sh
~/workspace$
```

```sh
$ date 
```

Jeder Befehl ht Optionen über die das Verhalten gesteuet werden kann.

```sh
$date -
```

## Optionen

- short-hand = 1 Buchstabe
- long-hand = Wort

```sh
$date --iso-8601
```

Quelle zu Imformationen über einen Befehl:

- häufig: Option ´--help´
- man pages (Manual = Handbuch) ´$ an date´, nicht in Replit, aber in gängigen Linux Distributionen
- Internetsuche mit "man date"
- KI

## Argumenente

Daten mit denen ein Befehlt arbeitet

```sh
$ echo Hallo Welt 
```

´Hallo Welt´ Ist hier das Argument.

mit Optionen

```sh
$ echo -n Hallo Welt
```

## Tastenkürzel

- Pfeil nach obe/unten: Historie
- Tabulator: automatische File Vervollständigung
- Ctrl-A: Anfang der Zeile
- Ctrl-E: Ende der Zeile
- Ctrl-C: Abbuch des aktuellen Kommndos

## File Befehle
-l -a
cd, mkdir, ls, touch, pwd , rm, rmdir 

Unterschied absoluter und relativer Pfad

Bedeutung von cd (ohne Argument) und von ~

### Bericht 
Absoluter Pfad:
- befindet sich in die Root-Verzeichnis und zeigt die genaue ort.
- Bsp: Du befindest dich in die Dokumenten verzeichnis. Es würde stehen c://Mario/documente/

Relativer Pfad: 
- Er zeigt nut teile des Pfad es zeigt nicht genau.
- Bsp: Du befindest dich in die Dokumenten verzeichnis. Es würde stehen Mario/documente/


cd: cd ist dafür das mann die directonary ändert.
cd(ohne Argument): es ändert der Direconary zum ganz anfang da wo sich alles 
befindet wichtig datein was admins oft zugriff haben.

mkdir: mkdir ist dafür um eine neue datei/ordner zu erstelle.

ls: inhalt von aktuelle folder.
ls-l: Gibt informationen üer ein Ordner

touch: Es ändert wer zugriff auf eine datei hat.

pwd: pwd zeigt wo du dich befindest.

rm: Kann eine datei löschen
rmdir: rmdir kann eine ordner und die inhalte löschen(sei vorsichtig damit).
